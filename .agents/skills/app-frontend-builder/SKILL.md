# App Frontend Builder

## Overview

Build interfaces that feel fast, legible, and intentional. Treat state transitions as part of the design, not an afterthought.

## Core Rules

1. Design loading, empty, error, and success states with the main screen.
2. Keep state close to where it is used unless sharing is necessary.
3. Prefer progressive disclosure over dense screens.
4. Optimize for keyboard access, touch comfort, and responsive layout.
5. Use motion to explain state change, not decorate randomly.

## Delivery Pattern

1. map the screen states
2. implement the layout shell
3. wire real data and interactions
4. add loading and error behavior
5. refine accessibility and responsiveness
