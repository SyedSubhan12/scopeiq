# App QA Release

## Overview

Test what matters to the user and what can break in production. Treat release readiness as evidence, not confidence.

## Core Rules

1. Test the primary user journey first.
2. Check loading, empty, failure, retry, and permission-denied states.
3. Validate contract edges, not just happy-path rendering.
4. Call out rollback or feature-flag needs when blast radius is high.
5. Distinguish must-fix regressions from acceptable follow-ups.

## Release Output

1. acceptance checklist
2. test coverage summary
3. known risks
4. release recommendation
5. post-release monitoring focus
