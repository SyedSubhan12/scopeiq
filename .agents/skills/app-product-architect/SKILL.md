# App Product Architect

## Overview

Translate a product ask into a shippable app shape. Prioritize clarity of flow, state boundaries, permissions, and sequencing over abstract architecture talk.

## Core Rules

1. Start from the primary user journey and critical task.
2. Define empty, loading, success, and failure states explicitly.
3. Keep navigation and information architecture shallow unless complexity is proven.
4. Split MVP scope from later enhancements.
5. Resolve permission and role boundaries before implementation begins.

## Deliverables

- main user journeys
- screen or route map
- feature boundaries
- states and transitions
- acceptance criteria
- MVP and follow-up split
